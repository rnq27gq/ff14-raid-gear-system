# FF14 零式装備分配システム

FF14の零式レイドチーム向けの装備分配管理システムです。8人固定チームでの公平で透明性のある装備分配を支援します。

## 概要

このシステムは8人レイドチームの装備分配を、公平な優先度アルゴリズムを使用して管理します。リアルタイムでの共同作業をサポートし、プレイヤーの希望を追跡し、長期的に公平な装備分配を実現します。

## 主な機能

- **Need/Greed/Passシステム**: 標準的なMMOの装備分配方式
- **スマート優先度計算**: ポジション優先度、装備方針、取得履歴を考慮
- **武器分配**: 武器箱と直ドロップ武器の専用処理
- **断章交換サポート**: 断章で交換した装備の高度な追跡機能
- **チーム認証**: パスワード保護によるセキュアなチーム作成
- **パスワード復旧**: セキュリティ質問による安全なパスワードリセット
- **リアルタイム同期**: チームメンバー全員へのライブ更新
- **取得統計**: 装備分配の包括的な追跡と履歴管理

## 利用方法

**アクセス**: [https://rnq27gq.github.io/ff14-raid-gear-system/](https://rnq27gq.github.io/ff14-raid-gear-system/)

### 新しいチームの作成

#### 1. チーム作成

1. アプリケーションにアクセス
2. 「新しいチームを作成」をクリック
3. 以下の情報を入力：
   - **チームID**: 3-20文字の英数字（他チームと重複不可）
   - **作成者名**: パスワードリセット時に表示される名前
   - **パスワード**: 6文字以上の安全なパスワード
   - **セキュリティ質問**: パスワードリセット用（5種類から選択またはカスタム）
   - **セキュリティ答え**: 質問の答え（忘れずに記録）

4. 「チーム作成」をクリック
5. 作成成功後、自動的にログインされます

#### 2. メンバー情報の設定

1. ダッシュボードから「メンバー管理」をクリック
2. 「メンバー情報」タブで8人全員の情報を入力：
   - プレイヤー名（必須）
   - ジョブ（ポジションに応じた選択肢から選択）

3. 入力完了後「メンバー情報を保存」をクリック

#### 3. 装備方針の設定

1. 「装備方針」タブに移動
2. 各プレイヤーの各部位（武器、頭、胴など）について方針を選択：
   - **零式**: 零式装備が適切な部位
   - **トームストーン**: トームストーン装備が適切な部位

3. 設定完了後「装備方針を保存」をクリック

#### 4. 武器希望の設定

1. 「武器希望」タブに移動
2. 各プレイヤーの武器希望を設定：
   - 第1希望: メインジョブの武器（自動設定）
   - 第2〜4希望: 他のジョブの武器を選択

3. 設定完了後「武器希望を保存」をクリック

### 装備分配の実行

#### 分配前の準備

1. ダッシュボードから対象の層をクリック（例：「1層分配」）
2. システムが自動的に分配優先度を計算

#### 分配の確認と実行

1. ドロップしたアイテムごとに推奨される分配先を確認
2. 必要に応じて分配先を変更：
   - プルダウンから別のメンバーを選択

3. 全てのアイテムの分配先を決定
4. 「分配を確定」をクリック

### 日常的な利用

#### 統計情報の確認

1. 「統計情報」から各メンバーの取得状況を確認
2. 公平な分配ができているかをチェック
3. 必要に応じて「編集開始」で手動調整

#### 分配履歴の確認

1. 「配布履歴」から過去の全分配記録を確認
2. 週別、アイテム別での絞り込み可能

#### ポジション優先度の調整

1. 「優先順位設定」でポジション間の優先度を調整
2. ドラッグ&ドロップで順序を変更
3. 「優先順位を保存」で確定

## 分配システムの仕組み

### 優先度計算

システムは以下の要素で分配優先度を自動計算します：

1. **装備方針**: 零式希望者 > トームストーン希望者
2. **ポジション優先度**: カスタマイズ可能（デフォルト: D1→D2→D3→D4→MT→ST→H1→H2）
3. **取得履歴**: 過去の取得数による動的調整
4. **武器希望**: 武器の場合は希望順位を考慮

### 断章交換システム

断章で装備を交換したプレイヤーの特別処理：

- **一時的除外**: 零式方針で未取得のプレイヤーがいる間は分配対象外
- **条件付き復帰**: 零式方針の未取得者がいなくなったら分配対象に復帰
- **自動ステータス更新**: 箱を取得すると「断章交換・箱取得済」に自動更新され完全に対象外

### 統計編集での断章交換設定

1. 「統計情報」→「編集開始」
2. 各プレイヤーの装備状況をプルダウンで選択：
   - 未取得
   - 取得済み
   - 断章交換
   - 断章交換・箱取得済

## チーム運用のコツ

### チームリーダー向け

- **定期的な統計確認**: 週次レイド後に取得状況をチェック
- **方針の見直し**: メンバーの装備状況に応じて装備方針を更新
- **データバックアップ**: 重要な進行時期にはデータをエクスポート

### チームメンバー向け

- **希望の明確化**: 装備方針と武器希望を正確に設定
- **リアルタイム確認**: レイド中はシステムにアクセスして分配状況を確認
- **履歴の把握**: 自分の取得履歴を定期的にチェック

## セキュリティとメンテナンス

### パスワード管理

- 強力で一意のパスワードを使用
- セキュリティ質問の答えは忘れずに記録
- メンバー変更時はパスワード更新を検討

### データ管理

- 重要な進行データは定期的にエクスポート
- チームIDは信頼できるメンバーのみと共有
- 問題発生時は統計編集機能で手動調整可能

## よくある問題と解決方法

### ログインできない

- チームIDとパスワードのスペル確認
- パスワードを忘れた場合は「パスワードを忘れた場合」からリセット
- ブラウザのJavaScriptが有効か確認

### データが更新されない

- ページを更新（F5キー）
- 他のタブで同じシステムを開いていないか確認
- インターネット接続の確認

### 分配結果がおかしい

- 装備方針の設定を確認
- 統計情報で過去の取得状況を確認
- 必要に応じて統計編集で手動調整

## 技術要件

- **対応ブラウザ**: Chrome, Firefox, Safari, Edge（最新版推奨）
- **インターネット接続**: 必須（リアルタイム同期のため）
- **JavaScript**: 有効化必須
